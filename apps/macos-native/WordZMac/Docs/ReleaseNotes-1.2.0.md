# WordZ 1.2.0 Release Notes

## 概览

WordZ 1.2.0 是一次以“研究可信度、性能和工程完成度”为核心的大版本升级。它不只是继续加功能，而是把原生 macOS 版本往真正可长期使用的语料分析工具又推进了一步。

这次更新主要围绕六条主线展开：

1. 分析可信度升级
2. 大语料性能专项
3. 语料库体系升级
4. 学术型分析能力扩展
5. 产品体验与 macOS 原生化打磨
6. 发布与工程化体系完善

## 主要更新

### 1. 分析可信度升级

- `Compare` 现在支持固定参考语料与自动主导语料两种模式。
- `Keyness / Log Ratio / p 值 / 差异` 会随参考语料真正重算，不再只是表面切换标题。
- `Compare` 当前多语料选择与参考语料会进入 workspace 持久化，恢复工作区时不再静默换成别的比较集合。
- `Stats / Word` 的 `Norm Frequency / Range / Rank / Norm Range` 已经成为统一统计列，并支持列显示控制与导出同步。
- 更多结果页导出时会自动附带分析参数、过滤条件和当前显示范围，便于复现实验。

### 2. 大语料性能专项

- `Stats / Word / Ngram / Compare` 大结果页继续采用阈值触发的后台 scene build，减少主线程卡顿。
- 仓储层新增分析结果缓存基础设施，并支持按分析类型定向失效。
- 语料信息读取改为优先走 `.db` 元数据与统计缓存，不再为了展示 `types / tokens / TTR / STTR` 重跑整篇分析。
- 表格层继续优化：密度预设、语义列宽、数值列专业排版、KWIC 特化布局和更稳的可见区刷新。

### 3. 语料库体系升级

- 本地语料库存储进一步稳定为真正的 SQLite `.db` 结构。
- 语料元数据从只读升级为可管理：
  - `source`
  - `year`
  - `genre`
  - `tags`
- 主工作区侧边栏与语料库窗口都能直接打开语料信息面板。
- 旧版文本和历史 `.db` 仍可继续读取，并会在需要时自动迁移到更稳定的结构。

### 4. 学术型分析能力扩展

- `Collocate` 新增三类更常见的关联指标：
  - `LogDice`
  - `MI`
  - `T-Score`
- 搭配分析页新增指标预设与“结果基于上次运行参数”的提示，降低参数修改后的误读风险。
- `KWIC / Locator` 增加更适合研究阅读的特化面板：
  - 固定宽度 concordance 行
  - 命中词高亮
  - 完整句查看
  - 复制引文
  - 从 KWIC 继续发送到定位器

### 5. 产品体验与 macOS 原生化打磨

- 主工作区继续收敛为分析中心，语料库管理、设置、任务中心、版本说明都作为独立窗口存在。
- 菜单、toolbar、右键动作和窗口入口进一步梳理，避免同一能力分散在多个位置而状态不同步。
- `Compare / Collocate / KWIC` 新增更明确的说明卡、空状态与详情面板，更适合学生、教师和研究者理解指标含义。
- 表格体验继续提升：
  - 紧凑 / 标准 / 阅读 三档密度
  - 默认列集按分析页定制
  - 数值列右对齐与等宽数字
  - 长文本移到详情区域，减少主表拥挤

### 6. 发布与工程化体系完善

- 原生打包流程补齐：
  - `build-app.sh`
  - `package-app.sh`
  - `release-manifest.sh`
  - `verify-release.sh`
  - `release-smoke.sh`
  - `release-checklist.sh`
- `manifest.json` 现在只保留可移植元数据，不再写入构建机绝对路径。
- `release-checklist.sh` 不再依赖打包脚本 stdout 的固定行号，发布脚本稳定性更高。
- 应用内“导出诊断包”升级为结构化 `.zip`，并默认对本地路径做脱敏处理。
- 诊断链路现在可以更清楚地区分版本、构建号、架构、commit、任务历史与当前工作区状态。

## 兼容性说明

- 历史工作区快照仍可继续恢复，缺失的新字段会自动回退默认值。
- 旧版 `Compare` 快照如果不包含多语料选择集，系统会保留当前有效选择并继续兼容固定参考语料恢复。
- 历史 workspace 里的 `Word Cloud` 标签仍可继续恢复，系统会自动回退到 `Word` 页签，避免旧快照失效。
- 历史 `.txt`、旧 JSON `.db` 与旧 SQLite `.db` 都可继续打开。

## 已知限制

- `Compare` 已具备固定参考语料，但更完整的 reference corpus 管理和批量实验输出仍会继续增强。
- 语料库 `.db` 已经成为真正底座，后续还会继续补索引、缓存和更强的查询能力。
- 诊断包目前默认脱敏路径；如果未来需要“支持模式下导出完整路径”，会作为显式选项加入，而不会默认开启。

## 版本策略

- 大功能更新继续使用 `1.3.0 / 1.4.0` 这类次版本号升级。
- 补丁修复沿当前次版本线递增，也就是 `1.2.1 / 1.2.2 / 1.2.x`。

## 适用人群

WordZ 1.2.0 更适合希望把本地语料分析真正用于教学、课程作业、论文研究和长期积累的学生、教师、研究者与普通文本工作者。
