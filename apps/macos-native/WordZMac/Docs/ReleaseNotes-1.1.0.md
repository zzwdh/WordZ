# WordZ 1.1.0 Release Notes

## 概览

WordZ 1.1.0 是原生 macOS 版本的一次大幅升级，重点围绕三件事展开：

1. 让本地语料导入、管理、分析和导出真正稳定可用。
2. 让统计结果、词表、KWIC、对比分析和表格阅读更接近正式产品，而不是技术预览界面。
3. 为后续继续扩展主题分析、参考语料比较、更多 NLP 能力和更大规模语料处理打好底层基础。

## 主要更新

### 1. 原生工作区与窗口体验升级

- 语料库管理从主工作区拆分为独立窗口，主页面只保留分析流程。
- 设置页拆分为独立窗口，避免设置状态干扰主工作区。
- 任务中心、版本说明和语料库入口统一纳入窗口体系，菜单结构更清晰。
- 主窗口、辅助窗口、菜单栏状态菜单和命令系统重新梳理，macOS 原生体验更稳定。

### 2. 本地语料库升级为 `.db` 存储

- 新导入语料现在保存为真正的 SQLite `.db` 文件，而不是仅使用 `.db` 后缀。
- 新增 `corpus_document`、`token_frequency` 等基础表，为后续检索、统计缓存和信息面板打底。
- 旧版 JSON `.db` 与历史文本存储会在读取时自动兼容并懒迁移。
- 语料修复逻辑增强，不再只检查文件是否存在，也会校验存储是否可读。

### 3. 分析能力增强

- 新增并稳定接入：
  - Chi-Square 卡方检验
  - Tokenize 分词视图
  - Topics 主题分析
  - Word Cloud 词云
- `Stats` 与 `Word` 新增：
  - `Norm Frequency`
  - `Range`
  - `Rank`
  - `Norm Range`
- `Compare` 由简单分布对比升级为带 `Keyness` 与 `Log Ratio` 的更严谨结果展示。
- `KWIC / Collocate / Locator / N-Gram` 全链路与工作区、导出、结果页同步增强。

### 4. 表格与结果阅读体验重做

- 结果表格新增三档密度：
  - 紧凑
  - 标准
  - 阅读
- 列宽不再靠列名猜测，改为基于列语义自动布局。
- 数值列统一右对齐、等宽数字与稳定的小数显示。
- `KWIC` 采用更适合语料检索的特化布局：左文、节点词、右文分区更清晰。
- `Compare` 与 `KWIC` 增加详情区域，长文本不再全部挤进主表。
- 不同分析页拥有更合理的默认列集，减少首屏信息噪音。

### 5. 性能与稳定性增强

- 大结果表的 scene build 改为阈值触发的后台构建，降低主线程阻塞。
- 搜索、停用词和词项规范化共用统一文本标准化逻辑，减少大小写、全角半角带来的结果偏差。
- 多个页面的 `apply(snapshot)` 改为批量状态恢复，避免重复触发 scene rebuild。
- 表格刷新从“整表重载”为主，优化为优先刷新可见区。
- 更新检查、自动下载、主题建模等后台流程加入并发保护，减少重复任务和状态污染。
- 工作区恢复、任务历史恢复、导出元信息和临时文件路径管理更稳。

### 6. 导出与学术使用体验增强

- `CSV / XLSX / Quick Look / Share` 统一走当前可见列。
- `XLSX` 导出支持元信息、冻结表头、列宽优化。
- 卡方结果正式接入统一导出链路，不再退回原语料文件。
- 词云 limit 改为纯展示与导出上限，调节时不需要重新分析。

## 兼容性说明

- 旧工作区快照会自动补默认值并继续加载。
- 历史 `.db` / 文本语料库可继续打开，系统会在需要时自动迁移到新的 SQLite 存储结构。
- 当前版本仍优先面向 macOS 原生版本，不引入额外第三方依赖。

## 已知限制

- `Compare` 已支持更严格的 keyness 展示，但完整的参考语料配置与更细粒度统计口径仍会继续增强。
- 更大规模语料下，SQLite 结构已经到位，但后续还会继续补索引和缓存策略。
- 主题分析目前仍以稳健本地方案为主，后续会继续打磨可解释性与参数复现能力。

## 适用人群

WordZ 1.1.0 适合学生、教师、研究者以及需要快速完成本地文本统计、检索、词表分析、对比分析和导出的普通用户。
