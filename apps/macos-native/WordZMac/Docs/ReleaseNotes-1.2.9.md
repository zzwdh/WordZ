# WordZ 1.2.9 Release Notes

## 概览

WordZ 1.2.9 是 `1.2.x` 线的一次稳定化发布，重点不在于继续堆叠零散页面，而是把原生 macOS 版本的研究工作流、语料库管理、更新体验和工程结构真正收紧到可长期迭代的状态。

这次更新主要围绕四条主线展开：

1. 语料库工作流升级
2. 研究工作流与结果留档升级
3. 更新与发布体验升级
4. 原生架构硬化与回归覆盖升级

## 主要更新

### 1. 语料库工作流升级

- 语料库窗口现在支持按 `来源 / 年份 / 体裁 / 标签` 进行元数据筛选。
- 新增“命名语料集”工作流，可以把当前筛选结果与语料成员保存为可复用集合。
- 选中多条语料后可以批量更新元数据，用于统一补 `source / genre / tags`。
- 语料库界面新增元数据完整性提示，能直接看出当前视图里缺年份、缺体裁、缺标签的数量。
- 语料信息、元数据编辑、批量维护与导入状态现在都集中在更稳定的原生窗口流里。

### 2. 研究工作流与结果留档升级

- 新增分析预设，支持保存、应用和删除当前分析页的工作区状态。
- 预设会保留标签页、搜索参数、语料选择和相关分析设置，方便复用同一套实验配置。
- 新增“研究报告包”导出，支持一次性打包：
  - 当前可见结果表
  - 方法摘要
  - 构建信息
  - workspace 草稿
  - 场景摘要文件
- 这让课堂作业、研究复核和结果回溯都更容易留档。

### 3. 更新与发布体验升级

- 原生版现在支持在应用内检查 GitHub Releases 最新版本。
- 启动时可按设置自动检查更新，并在发现新版本或检测到已下载安装包时直接进入更新窗口。
- 更新窗口会集中展示：
  - 当前版本
  - 目标版本
  - 安装包名称
  - 版本亮点
- 下载完成后可以直接从应用内打开安装包并重启，减少“看到新版本但不知道下一步做什么”的断点。

### 4. 原生架构硬化与回归覆盖升级

- 源码布局按 `Analysis / Workspace / Engine / Storage / Host / Export / Diagnostics / Shared` 等域重新收敛，减少旧 `Services/` 式堆叠。
- 主工作区、辅助窗口、菜单命令、场景同步和持久化职责进一步拆清，复杂状态切换更稳。
- 新增和补强了多组回归测试，覆盖分析预设、报告 bundle、更新流程、语料集、元数据批量编辑和结构守卫。
- 这次发布对用户最直接的价值不是“多一个入口”，而是更少状态漂移、更少窗口不同步，以及更稳的后续迭代基础。

## 兼容性说明

- 历史 workspace 快照仍可继续恢复，缺失字段会按当前默认值回退。
- 历史 `Word Cloud` workspace 标签仍可恢复，但会自动回退到 `Word` 页签以保持兼容。
- 现有本地语料、语料元数据和工作区持久化结构继续兼容，不需要手动迁移。

## 已知限制

- 当前自动更新链路仍基于 GitHub Releases，macOS 安装流程依然是下载后打开安装包完成替换。
- 批量元数据编辑目前仍以 `source / genre / tags` 为主，年份保持单条编辑更稳妥。
- 研究报告包优先导出当前可见结果与方法摘要，更重型的实验批量导出仍会继续增强。
