# WordZ macOS 右上角菜单栏图标计划

## 目标

- 在 macOS 顶部右侧提供一个常驻的 WordZ 入口。
- 让用户不用切回主窗口，也能完成最常用的轻量操作。
- 用菜单栏入口承接“后台任务状态、更新提醒、快速打开、快速预览”这类高频低成本动作。

## 不做什么

- 不在菜单栏里承载完整工作台。
- 不在菜单栏里展示大表格、复杂筛选器、长文本编辑器。
- 不在菜单栏里直接跑超重分析链路并阻塞主线程。

## 图标方案

### 视觉原则

- 使用单色 `template image`，自动适配浅色/深色菜单栏。
- 保持 16x16 视觉重心清晰，避免细碎文本。
- 第一版优先使用简洁图形，不直接塞 `WordZ` 全字样。

### 推荐方案

1. 主方案：自定义 `WZ` 抽象字形，两个字母合并成紧凑符号。
2. 备选方案：三条语料线 + 一个高亮检索点，表达“语料分析”。
3. 通知态：更新可用或任务运行时，在图标右下角叠加小圆点。

## 功能分期

### Phase 1：基础常驻入口

- 打开主窗口
- 导入语料
- 打开最近语料
- 打开设置
- 打开帮助
- 退出 WordZ

### Phase 2：工作区快捷动作

- 显示当前工作区名称
- 显示当前选中语料名称
- 快速预览当前语料
- 分享当前内容
- 打开本地语料库页

### Phase 3：后台状态与提醒

- 显示当前任务数
- 显示 Topics / 分词 / 统计等任务的进行状态
- 显示更新可用提示
- 一键打开更新说明或安装已下载更新

### Phase 4：轻量分析入口

- 对当前语料快速运行统计
- 对当前语料快速运行词表
- 对当前语料快速运行分词

说明：
- `Topics / Compare / KWIC / Collocate` 这类需要更多参数的分析，不建议第一版直接放进菜单栏。
- 菜单栏更适合“单击即做”的动作，不适合复杂配置流。

## 技术路线

## 推荐实现

- 使用 SwiftUI `MenuBarExtra`
- 最低系统已是 macOS 14，和当前 SwiftUI 原生架构一致
- 第一版采用菜单样式，优先稳定

### 结构建议

- `MenuBarExtra("WordZ", image: "MenuBarIcon")`
- 独立的 `MenuBarStatusViewModel`
- 从 `MainWorkspaceViewModel` 或共享 `AppCoordinator` 拉取只读状态
- 所有动作继续复用现有 `NativeAppCommandCenter` / `MainWorkspaceViewModel` 能力

### 为什么第一版不直接用 `NSStatusItem`

- `MenuBarExtra` 更贴合当前 SwiftUI 架构
- 代码量更小，状态同步更直接
- 足够覆盖菜单式入口需求

### 何时升级到 `NSStatusItem`

- 需要完全自定义 hover/动画
- 需要更复杂的富面板
- 需要更细粒度的高频状态刷新控制

## 信息架构建议

### 菜单栏一级内容

- 当前工作区
- 最近语料
- 快速操作
- 后台任务
- 更新与帮助

### 菜单示意

1. `WordZ`
2. 当前工作区：`Rose`
3. 当前语料：`Rose`
4. 分隔线
5. 打开主窗口
6. 导入语料
7. 打开本地语料库
8. 快速预览当前语料
9. 分享当前内容
10. 分隔线
11. 最近语料
12. 后台任务：`2 项进行中`
13. 检查更新
14. 使用说明
15. 设置
16. 退出

## 与现有能力的复用关系

- Quick Look：复用现有 `NativeQuickLookService`
- Share：复用现有 `NativeSharingService`
- 最近打开：复用 `settings.scene.recentDocuments`
- 更新：复用 `NativeUpdateService`
- 工作区动作：复用 `NativeAppCommandCenter`

## 风险点

1. 菜单栏入口和主窗口状态不同步
2. 后台任务过多时，菜单项会变长
3. 如果直接把重分析挂到菜单栏，容易造成“误触即跑”

## 对应规避

1. 菜单栏只读取共享状态，不维护第二份业务真相
2. 后台任务只显示摘要，不展示完整日志
3. 菜单栏中的分析功能只保留轻量、低参数、可立即执行的入口

## 实施顺序

1. 落地 `MenuBarExtra` 基础入口和模板图标
2. 接入“打开主窗口 / 导入语料 / 设置 / 帮助 / 退出”
3. 接入“当前工作区 / 当前语料 / 最近语料”
4. 接入 Quick Look / Share
5. 接入后台任务与更新状态

## 验收标准

- 菜单栏图标在浅色/深色模式下都清晰可辨
- 点击图标后 1 次交互内可打开主窗口
- 至少 5 个高频动作可在菜单栏完成
- 与主窗口状态同步，不出现错误语料名或错误任务数
