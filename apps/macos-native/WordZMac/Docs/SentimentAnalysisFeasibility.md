# WordZ 情感分析可行性报告

日期：2026-03-29  
适用版本：WordZ 1.1.0

## 1. 结论

在 WordZ 当前原生 Swift 架构下，英文情感分析是可行的，且适合分阶段推进。

推荐路线：

1. 第一阶段做英文规则/词典型情感分析 MVP。
2. 第二阶段引入可替换的本地分类模型接口。
3. 第三阶段再考虑段落级、句子级和语料对比级情感统计联动。

不建议当前版本直接上“重模型 + 多语言 + 高解释性”一体方案，因为这会同时抬高模型体积、推理时延、评测成本和 UI 复杂度。

## 2. 当前基础能力

WordZ 已具备几项非常适合承接情感分析的基础能力：

- 已有稳定的英文句子切分与英文词切分链路，基于 Apple `NaturalLanguage`。
- 已有本地分析引擎 `NativeAnalysisEngine`，适合继续承接轻量 NLP 任务。
- 已有主题分析、词频、KWIC、Collocate、Locator 等结果页，说明“分析结果 -> 表格 -> 导出”的产品路径已经成熟。
- 已有工作区状态持久化、表格导出、后台任务中心等能力，可直接复用。

这意味着情感分析不需要从 0 开始搭基础设施，主要工作集中在：

- 设计情感打分引擎
- 定义结果模型
- 做可解释的 UI
- 做英文语料评测与回归测试

## 3. 技术可行性判断

### 3.1 低风险可行

英文情感分析最容易落地的是规则/词典路线：

- 句子级切分后，按 token 匹配情感词典
- 增加否定词、程度副词、感叹号、对比连词等简单规则
- 输出 polarity、strength、positive hits、negative hits、neutral ratio

优点：

- 完全本地运行
- 推理速度快
- 体积小
- 结果可解释
- 非常适合学术演示和语料探索

缺点：

- 对讽刺、复杂长句、跨句语义理解较弱
- 领域迁移能力有限

### 3.2 中风险可行

第二条路线是接入本地文本分类模型：

- 句子级或段落级情感分类
- 输出 positive / neutral / negative 概率
- 可在后续做 corpus-level aggregation

优点：

- 效果上限更高
- 能处理部分词典法难覆盖的表达

缺点：

- 需要模型文件管理
- 需要评测集
- 首次集成成本显著高于规则法
- UI 需要解释“概率”和“置信度”，否则用户容易误解

### 3.3 高风险，不建议当前版本直接做

以下方案不建议在当前版本直接上：

- 多语言统一情感分析
- 大模型在线情感判别
- aspect-based sentiment analysis
- 细粒度讽刺/立场/情绪分类

这些方向会明显扩大产品范围，也会给本地性能、隐私策略、模型分发和测试带来额外压力。

## 4. 与 WordZ 产品形态的匹配度

WordZ 是英语语料分析软件，不是单条评论判别器。因此情感分析更适合做成“语料研究工具”，而不是“单句 AI 判断器”。

推荐输出层级：

1. 句子级：每句 polarity、score、触发词。
2. 段落级：段落平均分、极性分布。
3. 语料级：正向/负向/中性占比、平均情感强度、波动区间。
4. 对比级：不同语料或不同子语料的情感分布差异。

这与 WordZ 现有的：

- Stats
- Word
- Topics
- KWIC
- Compare

是高度兼容的。

## 5. 推荐的 1.1.x 实施方案

### 第一阶段：MVP

目标：2 到 4 周内做出可用版本。

建议功能：

- 新增 `Sentiment` 分析页
- 按句打分
- 输出 `sentence id / score / polarity / hits / sentence text`
- 支持筛选 `positive / neutral / negative`
- 支持导出 CSV / TXT
- 与 Locator 联动，点击句子可回看上下文

推荐算法：

- 英文情感词典 + 否定词规则 + 程度副词加权

理由：

- 实现快
- 易测试
- 可解释
- 足够验证用户是否真的需要该功能

### 第二阶段：增强版

在 MVP 被验证后，再补：

- 段落级聚合
- 文档级情感趋势图
- 与 Topics 联动，看每个 topic 的情感倾向
- 与 Compare 联动，看不同语料极性差异

### 第三阶段：模型化

若用户持续使用，再考虑：

- 可替换本地情感模型
- 领域词典
- 学术语料/新闻语料/社媒语料专用配置

## 6. 性能与稳定性影响

### 性能

规则型情感分析对 WordZ 当前架构非常友好：

- 可直接复用分词缓存
- 可直接复用句子切分结果
- 计算复杂度低
- 适合大语料批处理

预计热点不会在模型推理，而在：

- 大量句子结果的 UI 渲染
- 导出文件大小
- 多语料汇总统计

### 稳定性

当前最稳妥的方案是：

- 保持全本地执行
- 不依赖在线 API
- 词典与规则版本可控
- 输出明确的“规则型分析”标记

这样最符合 WordZ 当前“稳定优先、性能优先、可解释优先”的方向。

## 7. 主要风险

1. 用户会把“情感分数”误当成绝对结论，而不是启发式分析。
2. 英文学术语料、新闻语料、论坛语料的情感表达差异很大，单一词典会有偏差。
3. 否定、反讽、引用、转述会降低规则法准确率。
4. 如果直接上模型法，没有评测集就很难证明结果可靠。

## 8. 风险缓解建议

1. 在 UI 中明确标注“启发式英文情感分析”。
2. 默认展示触发词和规则命中，增强可解释性。
3. 支持导出句级结果，便于人工复核。
4. 为不同语料类型预留词典配置入口。
5. 在产品文案中避免使用“准确识别情绪”这种过强表述。

## 9. 建议优先级

优先级建议如下：

1. 先完成当前 `Tokenize` 能力和导出闭环。
2. 再做句级规则型 `Sentiment` MVP。
3. 最后再考虑主题-情感联动和模型化升级。

## 10. 最终建议

情感分析适合做，但应该按“规则型 MVP -> 结果验证 -> 再模型化”的顺序推进。  
对 WordZ 这种强调稳定性、性能和本地化的英语语料工具来说，这是风险最低、产出最高的路线。
